## Beispiel in cURL

Verwenden Sie ein Bearer Token im `Authorization` Header, damit ihre Anfrage basierend auf Ihren Rechten autorisiert wird. Ein Token können Sie im Serviceportal unter `/api` anzeigen lassen, nachdem Sie sich eingelogt haben.

**Beispiel: Inventardaten übermitteln für Bestellung Nr. 379**

```bash
curl -X POST "https://service-t.auctionline.ch/api/upload-data" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
  "ssp_order_id": 379,
  "u_type": "INV",
  "file": "YXNkZgo="
}'
```

<ParamField header="Authorization" param-type="string" required="true" deprecated="false">
  `Bearer YOUR_TOKEN`
</ParamField>

## Common mistakes

<Expandable title="Why am I getting a 401?" default-open="false">
  Verify that:

  - The token is valid and not expired.

  - The token is sent in the `Authorization` header.

  - The header value includes the `Bearer` prefix and a space before the token.
</Expandable>
