## Plattform-Jobs (cron)

| Cron-Frequenz         | Job                                         | Beschreibung                                                 |
| --------------------- | ------------------------------------------- | ------------------------------------------------------------ |
| `*/29 10-18 * * 1-6`  | <kbd>collect\_order\_confirmations</kbd>    | Bestellbestätigungen abholen                                 |
| `17 07,11,22 * * *`   | <kbd>get\_assigned\_users\_from\_m42</kbd>  | Meldung von Assigned Usern aus M42 abholen und verarbeiten   |
| `51 10,16,21 * * 1-6` | <kbd>backup</kbd>                           | Datenbank-Backup der Bewegungsdaten                          |
| `00 15 * * 1-6`       | <kbd>cleanup</kbd>                          | Löschen von bereits übermittelten DB Dumps älter als 20 Tage |

## Reporting

| Cron-Frequenz    | Job                                  | Beschreibung                                        |
| ---------------- | ------------------------------------ | --------------------------------------------------- |
| `59 */3 * * *`   | <kbd>load\_postgres</kbd>            | Bestellungen in Postgres-Tabellen laden für PowerBI |
| `00 07-22 * * *` | <kbd>load\_postgres\_articles</kbd>  | Bestellte Artikel laden mit aktuellem Status        |

### Reporting manuell start (Entwicklung / Debugging)

```bash
export db_host=$DB_HOST; /var/www/html/_systex/load_postgres.pl --db $DB_NAME --ext_table Orders_stage
```

## Hinweise

Eine an das DWH übermittelte Bestellung verleibt in der Postgres-DB unverändert; Aktualisierungen sind derzeit nicht vorgesehen.
